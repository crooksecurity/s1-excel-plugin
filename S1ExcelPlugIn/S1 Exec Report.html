Executive
Summary

SentinelOne unifies endpoint threat prevention, detection and response in a single platform driven by sophisticated machine learning and intelligent automation. The true value of the SentinelOne endpoint security system is the single point provision of the entire security triangle in a single endpoint agent that works autonomously while reporting to a single holistic management interface that provides access to all security functions.

The security triangle consists of:

PRE-EXECUTION: Using signature based technologies and our own deep file inspection static analysis, we prevent known-bad software (malware) from being executed.

ON EXECUTION: For those malware samples that bypass the prevention modules, and are able to achieve execution, we provide modules that detect unknown malware at the point of execution through the application of behavioral analysis.

POST-EXECUTION: Having detected unknown malware through its behaviors, we apply automated policy-based responses including mitigation (kill, quarantine, network quarantine) and provide remediation features. These, in conjunction with our real-time forensic analysis information, provide end-point security teams with all the information they need to restore end-points (servers, desktops and laptops) back to a known good working state.

This summary report provides some key performance indicators and metrics from the SentinelOne server.

Detected Threats

These are the detected threats and their statuses. They include both resolved and unresolved threats.

ACTIVE: These are threats that do not have a mitigation policy, or only have a "Kill" mitigation policy. These threats should be analyzed and further actions should be taken, such as Kill, Quarantine, or Rollback. Once an Action has been taken, acknowledge the notification to clear it out of the SentinelOne dashboard.

MITIGATED: These threats have been quarantined, remediated, or rolled back. Acknowledge the alert notification to clear them out in the SentinelOne dashboard.

BLOCKED: These threats have been prevented from executing based on blacklisted hash or static analysis (signature-less analysis). Acknowledge the alert notification to clear them out in the SentinelOne dashboard.

SUSPICIOUS: These are threats with suspicious behavior that should be analyzed further. If it is a false-positive, it should be marked as "benigh" and whitelisted. They can also be marked as a threat and further actions can be taken. Additional actions include Kill, Quarantine, and Rollback.

Detection Engines

SentinelOne uses different engines and techniques within the same agent to protect against malicious threats. This is a report of the different engines having convicted and detected the malicious activites.

Pre-Execution (Blacklist): As a first line of defense, the SentinelOne agents receive blacklisted signatures automatically from the server to protect against known threats. The immunization is automated to ensure protection against already discovered threats within the deployment environment.

Pre-Execution (DFI): The SentinelOne Deep File Inspection (DFI) is an intelligent, signature-less static prevention engine that is used to uncover and block known and unknown file-based malware. It leverages advanced machine learning algorithms instead of signatures. A key feature of DFI is the ability to provide protection even when the endpoint is offline without network connectivity.

Pre-Execution (Cloud Reputation): While the SentinelOne agent is capable of using DFI to protect against file-based malware, it is further enhanced with the capability of using aggregated Cloud intelligence to validate suspicions as needed or when appropriate.

On-Execution (DBT): This is the SentinelOne flagship Dynamic Behavioral Tracking (DBT) engine. Advanced threats are protected through cutting-edge behavior-based detection. DBT detects any type of malicious activity - from polymorphic malware to sophisticated exploits to stealthy insider attacks - against a full context of normal system activity.
